# Visualise Design MD Product Context

## Product Purpose

Visualise Design MD is a local-first browser tool for reviewing `design.md` files before implementation. It helps product builders, designers, and engineers inspect document structure, checklists, tables, raw Markdown, and verification issues without installing packages or uploading private files.

## Register

product

## Primary Users

- Product builders preparing design Markdown files for implementation.
- Designers documenting screens, components, states, interactions, and tokens.
- Engineers checking whether a design document is complete enough to build from.

## Core Jobs

- Load a Markdown design file locally.
- Understand the document structure quickly.
- Find unchecked verification items.
- Inspect tables in a readable format.
- Compare rendered Markdown with the raw source.
- Spot empty sections, duplicate headings, and other review issues.

## Product Principles

- Local first: no upload, no account, no network dependency.
- Fast review loop: drag file, inspect, fix source, reload.
- Static by default: one HTML file for version one.
- Verification over decoration: every visual element should help review quality.
- Portable: works from direct file open and can later deploy to Vercel.

## Tone

Clear, calm, practical, and precise. The interface should feel like a focused review bench, not a marketing site.

## Anti-References

- Busy dashboards with decorative charts.
- Developer portfolio aesthetics.
- Heavy dark-mode terminal UIs.
- Generic SaaS card grids.
- Overly playful Markdown previewers.
- Visual effects that distract from document review.
